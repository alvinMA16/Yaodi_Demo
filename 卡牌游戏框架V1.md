# Alibi Inquest

目前的计划是起名叫做 Alibi Inquest（缩写 AI），Roguelike 卡牌游戏，设定 PVE。主题是关于智能 AI 即将部署实装到社会上，从而引发了争端。游戏涉及三大利益方：政府（中性，会有摇摆）、大公司（推广 AI）、反 AI 组织（阻止 AI）。玩家扮演一个独立检察官，通过每天三次质询来推动故事发展。

体验目标：保守玩家、普通玩家偶尔暴富/暴雷、赌狗玩家把把梭哈。

## 一、目前基本定型的系统

### 1. 大框架与时间线

- 战局 = AI 上线前倒数 7 天，第 8 天结局结算。
- 每天流程大致固定：
  - 质询阶段：3 次质询机会。
  - 可以对同一人连问 3 次（深挖）。
  - 也可以三个人各问 1 次（广撒网）。
  - 通过质询抽取当日信息卡（质询结果）。
  - 白天可使用：携带的玩家卡 + 当天信息卡。
  - 日终结算：写报告 / 空报告（选择是否打出卡牌） -> 影响世界与压力。

### 2. 战局时间阶段划分

| 阶段 | 世界趋势 | 随机 | 玩家影响 |
| --- | --- | --- | --- |
| Day1-3 | 弱 | 弱 | 弱 |
| Day4-6 | 中 | 中 | 强 |
| Day7 | 中强 | 强 | 爆强 |
| Day8 | 锁定 | 无意义 | 无意义 |

### 3. 卡槽与仓库

- 玩家卡槽：3 个信息卡槽 + 3 个仓库携带卡槽。
- 每天抽到的新卡只能进入信息卡槽。
- 仓库卡槽仅储存带入战局的卡牌。
- 仓库容量 = 12（试运行）。
- 目标是防止“鼠鼠囤积流”或者“清空仓库直接退游”。

### 4. 压力系统

压力系统的设计目标是迫使玩家检视他们的行为，并且给予试图带卡出战局的玩家以压力。（顶级鼠鼠玩家会受到压力系统制裁）

#### 压力系统 V1

##### 一、压力变量

- `Pressure ∈ [0, +∞)`
- 初始 = `0`
- 无自然恢复机制

##### 二、每日结算公式

在每天结束时（Day 1-7）执行：

```text
P_next = P_prev
       + 留存惩罚
       + 销毁惩罚
       + 稀有使用惩罚
```

##### 三、留存惩罚（核心递增机制）

```text
留存惩罚 = L × Day² × 0.05
```

其中：

- `L` = 当天结束时手里保留的信息卡数量
- `Day` = 当前天数（1~7）
- `0.05` = 5% 系数

解释：

- `Day²` 使后期囤牌极危险
- 早期影响极小
- 囤牌者后期爆炸

举例：

- Day 2 留 2 张：`2 × 4 × 0.05 = 0.4`
- Day 6 留 2 张：`2 × 36 × 0.05 = 3.6`
- Day 7 留 3 张：`3 × 49 × 0.05 = 7.35`

后期压力明显变重。

##### 四、销毁惩罚

```text
销毁惩罚 = D × 3
```

- `D` = 当天主动销毁卡牌数量

特点：

- 强迫玩家不要疯狂抽满再扔
- 是一个立刻疼的惩罚

##### 五、稀有卡使用惩罚

- 蓝卡使用 = `+2`
- 红卡使用 = `+5`
- 白绿 = `0`
- 橙卡单独设计，不纳入基础公式

特点：

- 爆发打法更危险
- 红卡是明显高风险

##### 六、死亡判定

若 `P_next ≥ 120`，则死亡。

重要：

- 只在每日结算时判断
- 即使中途使用大量红卡，不会当场死亡
- 给予玩家“赌一把”的空间

## 二、战局结束时的五种结局类型

### 1. 强胜利（完美调查）

触发条件：

- 玩家存活
- 7 天都有提交报告（每天 >= 1 张卡打出）
- 玩家行为倾向 = 最终阵营排序匹配
- 不存在未完成报告
- 不存在死亡

匹配逻辑：

- 先计算玩家“行为倾向”
- 根据本局打出的卡，对 Gov / Corp / Anti 的净影响总和
- 最大值阵营 = 玩家倾向
- 若两个并列最大 -> 双倾向
- 若三方相等 -> 无倾向
- 再看最终阵营排序：
  - 单峰：`A > B > C`
  - 双峰：`A = B > C`
  - 三平：`A = B = C`

匹配规则：

- 单峰：若玩家倾向 = `A` -> 强胜利
- 双峰：若玩家倾向 ∈ `{A, B}` -> 强胜利
- 三平：直接判定强胜利（这种情况极难，直接给完美）

奖励：

- 抽卡奖励（后续待定）
- 可以带出未使用卡
- 无惩罚

### 2. 胜利（普通胜利）

触发条件：

- 玩家存活
- 7 天都有报告
- 玩家倾向 ≠ 最终主导阵营

也就是：

- 你完成了调查
- 但世界走向与你的“推动方向”不一致

奖励：

- 可带出未使用卡
- 抽卡奖励低于强胜利（具体未定）

### 3. 调查失败

触发条件：

- 玩家存活
- 存在“未完成报告”的天数
- 即某一天没有打出 >= 1 张卡

说明：

- 调查失败是惩罚玩家“不打卡”

结果：

- 随机 `-1` 卡
- 但不是死亡
- 仍然有结局（阵营排序仍然会生成）

### 4. 全空白

触发条件：

- 7 天完全没有提交报告
- 即从头到尾 `0` 张卡打出

说明：

- 每日空报告已经有压力惩罚
- 但“全空白”才应该触发严重掉卡惩罚

结果：

- 随机掉卡 6 张（基础）
- 视为彻底躺平玩法的制裁

### 5. 死亡

触发条件：

- 压力值 >= 死亡阈值
- 或其他未来特殊机制（目前 V1 只考虑压力）

结果：

- 随机掉卡 3 张（基础）
- 若携带橙卡惩罚翻倍 -> 可能 9 张
- 完全随机掏仓库 + 当局剩余卡

## 三、人物系统

### 1. 基础设定

#### 政府

- 政府发言人、技术主管、部长、副总统
- 政府发言人是默认人物
- 事态升级后会切换成技术主管
- 事态再次升级则会升级为部长
- 事态最严重时会换成副总统
- 政府端的标准触发程序为定档数值，危机时刻才会出现高级人物
- 起始刷率为：`90%、6%、3%、1%`

#### 公司

- 公司发言人、法务、高管、CEO
- 公司发言人是默认人物
- 变化数值变大后，升级为法务
- 变化数值变大后，升级为高管
- 变化数值超变大后，升级为 CEO
- 公司端的人物触发程序为单天巨大数值变化（无论升降）
- 单天数值变化巨大都会触发下一个人物
- 起始刷率为：`90%、6%、3%、1%`

#### 反对组织

- 游行者、领队、核心策划、反抗领袖
- 游行者是默认人物
- 综合参考值变化时，升级为领队
- 综合参考值变化时，升级为领队
- 综合参考值变化时，升级为领队
- 反对组织以政府和公司换人物数值为参考
- 当这两个派别的数值发生变化时，升级人物
- 起始刷率为：`90%、6%、3%、1%`

### 2. 阵营模型

#### 政府阵营模型

- 触发逻辑 = 危机层级 + 定档数值 + 稳定性
- 四层：
  - 政府发言人（默认）
  - 技术主管（危机升级）
  - 部长（高危）
  - 副总统（极危）
- 重点：政府触发是看“绝对数值”（Absolute Value）

#### 公司阵营模型

- 触发逻辑 = 单日波动 + 市场逻辑 + 危机公关
- 四层：
  - 公司发言人
  - 法务
  - 高管
  - CEO
- 公司触发看的是单日巨大变化（`ΔDay`）
- 这是市场行为模型的核心

#### 反对组织模型

结构：

- 游行者 -> 领队 -> 策划 -> 反抗领袖

定义变量：

- `G` = 政府绝对数值（稳定变量）
- `ΔC` = 公司单日变化数值（波动变量）
- `α` = 公司权重（建议 `1.5~3` 范围）
- `β` = 政府权重（建议 `0.5~1` 范围）

反对组织参考指数：

```text
R = α * |ΔC| + β * |G|
```

`|ΔC|` 使用绝对值是因为：

- 爆涨 -> 推动 AI -> 抗议
- 爆跌 -> 风险增加 -> 抗议

升级触发建议：

- `R >= 20` -> 领队
- `R >= 35` -> 策划
- `R >= 50` -> 反抗领袖

三阵营触发统一框架：

| 阵营 | 观察变量 | 模式 | 行为意义 |
| --- | --- | --- | --- |
| 政府 | G（绝对值） | 稳定 -> 危机 | 官僚政治 |
| 公司 | ΔC（日变化） | 波动响应 | 市场政治 |
| 反对 | αΔC + βG | 风暴响应 | 群体政治 |

## 四、阵营基础结构与升级机制

### 1. 阵营基础结构

- `Gov = 0`
- `Corp = 0`
- `Anti = 0`

允许极小随机扰动（可选），但核心基准为 `0`。

- 数值可以正负波动。
- 允许超过 `±100`，不裁剪（否则会破坏政府绝对值模型）。

### 2. 人物升级机制（核心系统）

三阵营不同触发逻辑。

#### 1. 政府模型（稳定危机模型）

观察变量：

- `G = |Gov|`

升级区间：

| 层级 | 人物 | 区间 |
| --- | --- | --- |
| L1 | 发言人 | 0-19 |
| L2 | 技术主管 | 20-39 |
| L3 | 部长 | 40-69 |
| L4 | 副总统 | >=70 |

跳级机制：

- `|ΔGov| >= 30` -> 至少 L2
- `|ΔGov| >= 60` -> 至少 L3
- `|ΔGov| >= 90` -> 直接 L4

特点：

- 慢升
- 看长期绝对值
- 国家层面危机模型

#### 2. 公司模型（市场波动模型）

观察变量：

- `ΔC = 当日 Corp 变化绝对值`

升级区间（单日判定）：

| 层级 | 人物 | 条件 |
| --- | --- | --- |
| L1 | 发言人 | - |
| L2 | 法务 | >=10 |
| L3 | 高管 | >=20 |
| L4 | CEO | >=35 |

特点：

- 只看单日
- 不看累计
- 市场应激模型

#### 3. 反对组织模型（风暴响应模型）

变量：

- `G = |Gov|`
- `ΔC = |当日 Corp 变化|`
- `α = 2.0`
- `β = 0.7`

计算：

```text
R = α|ΔC| + β|G|
```

升级区间：

| 层级 | 人物 | 条件 |
| --- | --- | --- |
| L1 | 游行者 | R < 20 |
| L2 | 领队 | >=20 |
| L3 | 策划 | >=35 |
| L4 | 领袖 | >=50 |

特点：

- 公司权重大
- 政府长期极端也会积累火药桶
- 风暴模型

### 4. 均衡性说明

- 阶段门控保证节奏
- Day1-3：全阵营几乎无红橙（风暴也不爆）
- Day4-6：红开始成体系，橙只在高等级小幅出现
- Day7：想要的“神选/地狱”才真正成立
- 阵营风格保留
- 公司：蓝红橙在后期/高等级明显更高（爆发）
- 政府：红橙永远最克制（稳慢）
- 反对：红橙在后期非常凶（风暴）
- 不会把 V1 推成“红橙满天飞”
- 橙卡太多会毁掉 V1
- 橙的爆发主要锁在 Day7 + L4，而不是全程泛滥

## 六、结局

| 排序 | 结局 | 说明 |
| --- | --- | --- |
| 政府 > 公司 > 反对 | 《合法合规》 | AI 受损上线，政府指导公司，并且负有监督管理权，反对组织失败 |
| 政府 > 反对 > 公司 | 《国有资产》 | AI 受损上线，政府听取了反对组织意见，实施了国有化 |
| 政府 = 反对 > 公司 | 《隐私保护》 | AI 暂停上线，政府全面审查 AI 是否尊重人民的隐私，反对组织全程监督，公司失败 |
| 政府 = 公司 > 反对 | 《民生第一》 | AI 受损上线，政府选择有限度地支持公司，并且提出了税收、就业要求，反对组织失败 |
| 政府 > 反对 = 公司 | 《明天再说》 | AI 延缓上线，公司与反对组织各执一词，政府选择等待结果 |
| 反对 > 政府 > 公司 | 《拦腰斩断》 | AI 未能上线，政府听取了反对组织以及舆论意见，直接立法禁止了 AI 部署 |
| 反对 > 公司 > 政府 | 《黑客帝国》 | AI 叛乱上线，反对组织渗透公司上传恶意代码，公司为了保证市值并未直接下线，政府无作为 |
| 反对 > 政府 = 公司 | 《人民万岁》 | AI 受损上线，但是遭到了人民的抵制，政府和公司互相制衡 |
| 公司 > 反对 > 政府 | 《无伤大雅》 | AI 受损上线，公司成功打压反对组织，少数人选择抛弃 AI，但是大多数用户都被留下了，政府无作为 |
| 公司 > 政府 > 反对 | 《公司至上》 | AI 完整上线，政府完全支持公司，反对组织失败 |
| 公司 = 反对 > 政府 | 《握手言和》 | 公司选择与反对组织达成和解，AI 受限上线，政府无作为 |
| 公司 > 政府 = 反对 | 《坐收渔利》 | AI 完整上线，政府和反对组织来回拉扯，公司趁机上线成功 |
| 政府 = 公司 = 反对 | 《三权分立》 | 三方陷入了胶着，现实没有发生变化 |

解锁所有 13 个常规结局达成成就：《这不太吉利》（常规结局，不包含未来特殊路线）

## 八、橙卡总表（试运行）

规则：

- 单局最多携带 1
- 战局内最多出现 3
- 超过不再刷新
- 禁止携带、打出同款橙卡（携带即不刷，打出后再重新计算刷率）

### 1. 橙卡「梭哈」

- 效果：第 7 天，当日所有阵营的数值变化结算 ×2（包括玩家打牌 + 世界自动波动）
- 惩罚：本局死亡时 -> 掉仓库卡牌数量 ×3（从 3 张变成 9 张）
- 注释：非常纯粹的 Day7 赌狗按钮

### 2. 橙卡「押宝」

- 效果：立刻让你选定的阵营刷出最高等级人物（政府 = 副总统，公司 = CEO，反对 = 反抗领袖），从当日质询开始生效
- 惩罚：当日结束时，如果这个阵营的总数值不是三方中最高 -> 直接死亡
- 注释：“你敢押这边当老大？没当成，你就死。”

### 3. 橙卡「盘口」

- 效果：
  - 在打出这张牌的当天结束时，随机选中一个阵营（Gov / Corp / Anti，等概率）
  - 计算这一天该阵营的总变化量（今天所有加减的合计）
  - 结算时，这个阵营的当日变化量 ×2
  - 另外两个阵营的当日变化量 ×0.5（可向下取整）
- 惩罚：没有额外惩罚，惩罚就是它是随机阵营
- 注释：“可以赌一把自己的阵营会飞天，也可能直接坠入地狱。”

### 4. 橙卡「禁牌」

- 效果：当天你选定一个阵营，该阵营在这一天的结算规则为“无论今天怎么算，最终当日净变化一定视为负值”
- 惩罚：当日结束后立刻增加玩家压力 `+25`
- 注释：压力换胜利，拿命换暴力

### 5. 橙卡「政变」

- 效果：最高阵营和最低阵营的最终数值互换（全套交换，而不是只动一点）
- 用的时候最好是“站在落后方”，用这张卡翻盘
- 惩罚：只要不是“完美调查结局”，一律死亡
- 注释：我不入地狱，谁入地狱

### 6. 橙卡「双杀」

- 效果：本日结算时，你可以宣告两个阵营必须同涨同跌
- 惩罚：若最终这两个阵营中的任意一个在最终结局落败，则玩家直接死亡
- 注释：从小就穿一条裤子

### 7. 橙卡「舆情」

- 效果：打出后，在本局剩余所有天数里，玩家信息卡结算 ×2
- 惩罚：在剩余所有天数里，不得留存任何卡牌，每日必须打出所有卡牌，否则直接死亡
- 注释：今日事，今日毕

## 九、明确缺失的系统

1. 世界自动系统：特殊事件、阵营行为、强度矫正等等
2. 文字系统：故事、卡牌信息、战局信息等
3. 任务线：推动特定结局以构筑剧情 + 特殊奖励；每日任务给予抽卡奖励等
4. 抽卡奖励机制：如何根据用户的情况给予一定程度的抽卡参数调整
5. 数值模拟系统：Alpha 阶段内部测试，用以检测游戏平衡以及特殊情况是否会击穿战局
