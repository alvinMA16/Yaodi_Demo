# Alibi Inquest Demo 使用说明

这是一个本地运行的卡牌调查游戏原型项目，里面有 3 个部分：

- 玩家页面：用来实际游玩 demo
- 管理后台：用来调整数值、发布版本、查看对局记录
- 本地接口服务：给前面两个页面提供数据

如果你只是第一次接触这个项目，可以把它理解成：

- `http://localhost:5173` 是“游戏页面”
- `http://localhost:5174` 是“管理页面”
- `http://localhost:3001` 是“后台服务地址”，一般不用手动打开

## 你需要先准备什么

在开始前，请先安装下面这 2 样东西：

1. Node.js
2. npm

通常安装 Node.js 后，`npm` 会一起安装好。建议直接安装 Node.js 官网提供的 LTS 版本。

安装完成后，打开终端，在项目根目录执行下面命令：

```bash
npm -v
node -v
```

如果能看到版本号，说明环境已经准备好了。

## 第一次启动

下面的命令都要在项目根目录执行，也就是 `README.md` 所在的这个文件夹。

### 1. 安装依赖

```bash
npm install
```

这一步会下载项目运行需要的工具。第一次执行可能会稍微久一点。

### 2. 创建环境变量文件

把 `apps/api/.env.example` 复制一份，命名为 `apps/api/.env`。

如果你用的是 macOS 或 Linux，可以直接执行：

```bash
cp apps/api/.env.example apps/api/.env
```

如果你不想用命令，也可以手动复制这个文件并改名。

### 3. 启动项目

```bash
npm run dev
```

看到服务启动成功后，保持这个终端窗口不要关闭。

### 4. 打开页面

在浏览器里打开：

- 玩家页面：`http://localhost:5173`
- 管理后台：`http://localhost:5174`

## 玩家页面怎么用

打开 `http://localhost:5173` 后，可以按下面流程体验：

1. 点击“新开一局”
2. 在“质询”区域点击 `gov`、`corp` 或 `anti`，抽取信息卡
3. 对抽到的卡执行“打出”，或者勾选“日终销毁”
4. 在“日终处理”里点击“提交报告并结束当天”或“空报告结束当天”
5. 重复上面的操作，直到战局结束

页面里你会看到这些关键信息：

- `Day`：当前是第几天
- `Pressure`：当前压力值
- `Inquiries`：当天还剩多少次质询机会
- `World`：三个阵营当前的推进情况

## 管理后台怎么用

打开 `http://localhost:5174` 后，可以做这几件事：

1. 查看当前有哪些数值版本
2. 点击“基于当前版本创建草稿”生成一个新草稿
3. 修改名称、描述和 JSON 配置
4. 点击“保存草稿”
5. 确认没问题后，点击“发布为激活版本”
6. 在下方查看最近的对局记录和操作日志

简单理解就是：

- 玩家页面负责“玩”
- 管理后台负责“改规则、看记录”

## 数据保存在哪里

这个 demo 的本地数据会保存在：

`apps/api/data/store.json`

这意味着你在玩家页面开的局、在后台创建的版本，都会写进这个文件。

## 关闭项目

在启动项目的终端窗口里按 `Ctrl + C` 就可以停止运行。

## 常见问题

### 1. 页面打不开

先确认你有没有执行：

```bash
npm run dev
```

如果执行了，看看终端里有没有报错。

### 2. 提示端口被占用

默认端口是：

- 玩家页面：`5173`
- 管理后台：`5174`
- API：`3001`

如果这些端口已经被别的程序占用，项目可能无法正常启动。先关闭占用这些端口的程序，再重新执行 `npm run dev`。

### 3. 改了后台配置，但玩家页没变化

请确认你已经在管理后台点击了“发布为激活版本”。只有发布后的版本，才会在新开一局时生效。

### 4. 想重新开始测试

先停止项目，再删除本地数据文件：

`apps/api/data/store.json`

下次启动时，系统会重新生成本地数据。
